# MHDSRA2 vs DSRA 论文式结论摘要

## 中文摘要

本轮基准在统一随机输入、统一序列长度网格与统一槽位配置下，对 MHDSRA2 与基线 DSRA 的流式推理效率和状态内存开销进行了系统比较。实验覆盖 `seq_len=131072~1048576`、`slots={64,128,256}`、`read_topk={4,8}` 与 `chunk_size={512,1024,2048,4096}` 共 `120` 组案例。在 `batch_size=1`、`dim=64`、`retrieval=False` 的当前设置下，MHDSRA2 在 `79/120` 组案例中快于 DSRA，占比 `65.8%`；其平均加速比为 `1.287x`，中位加速比为 `1.337x`，说明多头化与局部门控机制在大多数长上下文场景下已表现出稳定的吞吐优势。

进一步地，性能收益呈现出明确的结构性规律。随着 `slots` 从 `64` 增长到 `256`，MHDSRA2 的平均加速比由 `1.062x` 提升至 `1.570x`，表明其更适合在较大记忆槽规模下发挥优势。`chunk_size` 对结果影响最为显著：当 `chunk_size=512` 时，MHDSRA2 平均仅为 `0.875x`，尚未超过基线；而当 `chunk_size=2048` 时，平均加速比升至 `1.728x`，成为当前最优工作区间；继续增大到 `4096` 后仍可维持 `1.511x` 的较高收益。这说明 MHDSRA2 的实现更适合与中大块流式切分策略配合使用。

在状态内存口径上，本次已按“MHDSRA2 相对 DSRA 的状态开销倍率”重新定义统计语义，避免旧版 `DSRA/MHDSRA2` 比值在“最佳/最差内存案例”命名上造成歧义。修正后可见，MHDSRA2 的平均状态开销为 DSRA 的 `1.072x`，即平均多占用约 `7.2%` 的流式状态内存。最优案例下，该倍率为 `1.009x`，几乎与 DSRA 持平；最差案例下为 `1.238x`，额外开销约 `23.8%`。从分组结果看，`chunk_size=2048/4096` 时内存倍率分别为 `1.041x/1.021x`，说明在性能优势最明显的区间，MHDSRA2 的额外状态成本实际上较为温和，具备较好的工程可接受性。

综合来看，MHDSRA2 已在长序列、高槽位、中大块配置下展现出“以有限状态开销换取显著吞吐收益”的明确趋势。当前结果支持如下判断：其一，MHDSRA2 在长上下文流式建模中具备替代或补充 DSRA 的现实潜力；其二，最佳部署区间并非小块切分，而是 `chunk_size>=2048` 的中大块配置；其三，现阶段结论仍主要反映“随机负载下的效率-内存权衡”，尚未覆盖显式检索开启后的准确率收益，因此仍需在下一轮基准中引入 retrieval 场景与任务级指标，以完成对“效率、内存、效果”三者关系的闭环评估。

## 汇报版结论

- 结论一：在当前 `120` 组长上下文案例中，MHDSRA2 有 `65.8%` 的情形快于 DSRA，平均加速 `1.287x`。
- 结论二：MHDSRA2 的优势主要出现在较大 `slots` 与较大 `chunk_size` 区间，尤其是 `chunk_size=2048` 时平均加速达到 `1.728x`。
- 结论三：修正统计口径后，MHDSRA2 的平均状态内存为 DSRA 的 `1.072x`，额外开销总体可控，不应再用旧版“best/worst memory case”直接表述。
- 结论四：当前最佳工程区间可概括为“更大槽位 + 中大块切分”，而非“小块 + 低槽位”。
- 结论五：下一轮必须补上 `retrieval=True`、更大 `dim/batch_size` 与任务准确率对比，才能形成完整的论文式论证链条。

## 下一轮基准设计

- 目标一：验证 `retrieval=True` 后，MHDSRA2 是否能用可接受的延迟与状态开销，换取更好的长程召回准确率。
- 目标二：验证在更大模型宽度和更高并发批量下，当前吞吐优势是否仍然成立。
- 目标三：将“吞吐、状态内存、任务准确率”合并为同一张比较表，形成更接近论文实验部分的证据结构。

### 建议实验矩阵

| 维度 | 建议取值 | 目的 |
|---|---|---|
| `use_retrieval` | `False`, `True` | 区分纯流式机制与带检索机制的增益 |
| `retrieval_tokens` | `32`, `64`, `128` | 观察检索读回规模对时延和准确率的影响 |
| `batch_size` | `1`, `2`, `4` | 观察并发提升后的吞吐与状态扩张 |
| `dim` | `64`, `128`, `256` | 观察模型宽度扩大后的扩展性 |
| `slots` | `128`, `256` | 聚焦当前已表现较优的槽位区间 |
| `chunk_size` | `1024`, `2048`, `4096` | 聚焦当前有效区间，避免在明显劣势区间浪费成本 |
| `seq_len` | `131072`, `262144`, `524288`, `1048576` | 保留长上下文代表性 |

### 建议任务指标

- `Needle-In-A-Haystack`：记录最佳命中率或固定训练预算下的命中率，衡量远距离关键信息召回能力。
- `JSON Retrieval`：
  - `teacher_forced_exact_match_rate`
  - `teacher_forced_mean_sequence_accuracy`
  - `generation_exact_match_rate`
  - `generation_mean_sequence_accuracy`
- 效率指标：
  - `elapsed_ms`
  - `tokens_per_second`
- 内存指标：
  - `MHDSRA2/DSRA state bytes ratio`
  - 必要时补充运行峰值显存
