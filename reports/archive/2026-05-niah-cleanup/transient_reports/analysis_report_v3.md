## 🚨 问题诊断与修复

### 问题 1: Phase 2 结果为什么比 Phase 1 差？

**诊断结论**: 发现严重 Bug！

#### 根本原因

Phase 2 的 `phase2_ablation_test()` 函数**没有在测试每个模型前重置随机种子**：

```python
# Phase 1 (正确):
for cfg in configs:
    set_seed(SEED)  # ✅ 每个配置前重置随机种子
    model = HybridModel(...)
    train_and_evaluate(...)

# Phase 2 (错误):
print("\n--- Pure ST (2 layers) ---")
model_st = PureST(...)
train_and_evaluate(...)  # ❌ 没有重置随机种子！

print("\n--- Pure MHDSRA2 (2 layers) ---")
model_mh = PureMH(...)
train_and_evaluate(...)  # ❌ 没有重置随机种子！
```

#### 影响分析

1. **数据生成不一致**: 每个测试使用不同的随机序列
2. **模型初始化不同**: 每个测试使用不同的随机权重
3. **训练扰动不同**: 每个测试的梯度噪声不同

这导致了**不可比的结果**！

#### 修复方案

在 Phase 2 的每个测试前添加 `set_seed(SEED)`：

```python
print("\n--- Pure ST (2 layers) ---")
set_seed(SEED)  # ✅ 修复：重置随机种子
model_st = PureST(...)
train_and_evaluate(...)

print("\n--- Pure MHDSRA2 (2 layers) ---")
set_seed(SEED)  # ✅ 修复：重置随机种子
model_mh = PureMH(...)
train_and_evaluate(...)
```

#### 验证结果

创建了诊断脚本 `scripts/diagnose_phase2.py`，验证随机种子功能正常。

**预期修复后的结果**：
- Pure ST 应该接近 Phase 1 的 Baseline (25.0%)
- Pure MHDSRA2 应该接近 25.0%
- Hybrid (Weighted) 应该接近 85.0%

### 问题 2: 测试长度太短，32K 不适用

**诊断结论**: 需要完整的 2M 测试。

#### 原始测试的局限性

| 测试 | 序列长度 | 问题 |
|------|----------|------|
| Phase 1-2 | 2048 | 太短，无法验证流式能力 |
| Phase 3 (旧) | 32K | 仍然不够，真实场景通常 64K-2M |
| Phase 4 | 4096 | 位置编码越界 |

#### 2M 测试设计

**测试策略**: 逐步增加序列长度

```
32K → 64K → 256K → 2M
```

**配置调整**:

| 序列长度 | Chunk Size | 训练步数 | 评估间隔 | 评估批次 |
|----------|-----------|---------|---------|---------|
| 32K | 4096 | 200 | 50 | 5 |
| 64K | 4096 | 200 | 50 | 5 |
| 256K | 8192 | 100 | 50 | 3 |
| 2M | 16384 | 50 | 25 | 2 |

**关键优化**:
1. **Chunk Size 自适应**: 随序列长度增加
2. **训练步数递减**: 避免过长的训练时间
3. **评估批次递减**: 减少评估开销
4. **早停机制**: 如果 32K 失败，跳过后续测试

## 🔍 深入分析

### 2M 测试的准确率分析

#### 为什么 256K 准确率降到 0%？

**可能原因**:

1. **训练步数不足**: 100 步对 256K 序列不够
   - 256K/8192 = 32 个 chunks
   - 每个 step 只更新一次，梯度信息有限
   
2. **评估过于稀疏**: 只在最后评估，可能错过学习过程

3. **梯度消失**: 长序列导致梯度传播困难
   - 32 个 chunks 的梯度累积可能导致数值不稳定

#### 改进建议

```python
# 增加训练步数
if seq_len <= 65536:
    steps = 500  # 从 200 增加到 500
elif seq_len <= 262144:
    steps = 300  # 从 100 增加到 300
else:
    steps = 100  # 从 50 增加到 100

# 增加学习率
lr = 1e-3  # 从 5e-4 增加到 1e-3

# 增加评估频率
eval_interval = 50  # 每 50 步评估一次
```
